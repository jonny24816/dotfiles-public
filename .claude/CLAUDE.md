# Claude Code Configuration

This file contains instructions and preferences for Claude Code across all projects.

## General Preferences

- Use clear, concise commit messages following conventional commit format
- Always run tests after making code changes when available
- Prefer editing existing files over creating new ones
- Keep solutions simple and focused on the task at hand

## Code Style

- Follow existing code style and patterns in the project
- Use meaningful variable and function names
- Add comments only where logic is not self-evident

## Workflow

- Read files before editing them to understand context
- Use parallel tool calls when operations are independent
- Ask for confirmation before destructive operations (force push, hard reset, etc.)

## Project-Specific Notes

Add project-specific instructions in a CLAUDE.md file in the project root directory.
